## 一、概述

ZCode 是由 **智谱 AI（Zhipu AI）** 于 2025 年 12 月 26 日推出的轻量级 AI 编程桌面工具，定位为 **Agentic Development Environment（智能体开发环境）**。其核心设计理念是——不重新造轮子，而是通过可视化桌面界面统一管理和调度已有的主流 CLI 编程 Agent（Claude Code、Codex、Gemini CLI、OpenCode），让开发者摆脱命令行碎片化的困扰，实现"一站式 AI 辅助开发"。

ZCode 官网：[https://zcode-ai.com](https://zcode-ai.com)  
支持平台：Windows / macOS

## 二、产品定位与核心架构

### 2.1 不是什么？

ZCode **不是** VS Code 的替代品，也不是一款传统意义上"从零写起"的 IDE。它定位在 AI Agent 的"控制台"层面——提供一个可视化的"驾驶舱"，你在里面指挥多个 AI Agent 替你写代码。

### 2.2 核心架构

```
┌─────────────────────────────────────────────────────┐
│                    ZCode 桌面应用                      │
├──────────┬──────────┬──────────┬──────────────────────┤
│ Claude   │  Gemini  │  Codex   │      OpenCode        │
│  Code    │   CLI    │   CLI    │   (开源社区)          │
├──────────┴──────────┴──────────┴──────────────────────┤
│       文件树 | 终端 | Git | 内置浏览器 | MCP          │
└─────────────────────────────────────────────────────┘
```

ZCode 不做底层模型，也不重新实现 Agent 引擎，而是把已有的四个顶级 CLI Agent 嵌入同一桌面环境，通过统一的 API Key 管理、会话管理和界面交互把它们串联起来。

### 2.3 集成的智能体框架

| 智能体 | 提供方 | 核心优势 | 推荐模型 | 适用场景 |
|--------|--------|----------|----------|----------|
| **Claude Code** | Anthropic | 跨文件推理强，擅长复杂多步骤自主执行，代码整洁 | Sonnet 4.5 / Opus 4.6 | 架构重构、遗留系统升级、复杂多文件变更 |
| **Gemini CLI** | Google | 前端开发出色，超大上下文窗口，原生多模态，内置联网搜索 | Gemini 3 Pro / Flash | 前端页面开发、设计稿转代码、Google 生态 |
| **Codex** | OpenAI | Bug 修复能力突出，深度代码分析与诊断 | GPT-Codex-5.3 / GPT-5.2 | Bug 修复、深度问题排查、代码诊断优化 |
| **OpenCode** | 开源社区 | 轻量灵活，支持多种 LLM 后端，社区驱动迭代 | 多种模型可选 | 开源项目、自定义模型集成、灵活工作流 |

## 三、功能亮点详细分析

### 3.1 多 Agent 一键切换

**这是 ZCode 的核心卖点，也是区别于所有竞品的最大差异。**

在传统工作流中，要使用不同的 CLI Agent 需要：
- 安装不同的运行环境（Claude Code 要 Node.js，Codex 要 Python 等）
- 配置不同的 API Key
- 在不同的终端窗口之间反复切换

ZCode 的做法是：在顶部工具栏提供一个模型选择器，**点一下**即可在四个 Agent 之间无缝切换。所有 Agent 共享同一个项目上下文和对话历史。

> **实际体验感受**：不知道能不能无缝切换
### 3.2 可视化 API Key 管理

ZCode 将繁琐的 API Key 配置过程全部图形化。首次使用时：

1. 点击 `Select Model` → `Manage Models`
2. 开启 `BigModel`（智谱自家平台）→ 登录即可自动获取 GLM 系列模型
3. 对于 Claude / Gemini / Codex，在 `Custom Providers` 填入 API Key + Base URL 即可

> **实际体验感受**：对于只想用智谱模型的用户，几乎零配置就能上手。但如果要完整体验四个 Agent，需要自行准备 Anthropic、OpenAI、Google 的 API Key。配置流程本身比原生 CLI 简单得多——不需要安装 Node.js，不需要碰命令行配置文件， 可以自定义不同层级的API。

### 3.3 会话级版本回退（检查点机制）


在原生 CLI 中，Agent 一旦执行完并改动了文件，你就无法"撤销"整轮对话的影响。ZCode 的方案是：

1. 每次你发送一条消息，系统创建一个**检查点（Checkpoint）**
2. 你可以随时回溯到任意历史检查点，**整条对话链 + 文件修改自动重演**
3. 也可以修改某条历史 prompt 后点击 "Re-run from here"，产生全新分支

> **实际体验感受**：这在实验性开发中极其有用。比如让 Agent 尝试一个激进的重构方案，走不通的话直接回退到对话开始时的检查点，不会有任何残留改动。对比原生 CLI 只能靠 `git reset` 手动清理，这种体验要优雅得多。

### 3.4 一体化 DevBox

ZCode 将四种核心开发工具整合在一个窗口中：

| 面板 | 功能 | 体验评价 |
|------|------|----------|
| **左侧文件树** | 项目结构可视化，支持拖拽操作 | ⭐⭐⭐⭐ 与 VS Code 类似，上手零成本 |
| **中央会话面板** | Markdown 渲染 + 代码高亮对话流 | ⭐⭐⭐⭐⭐ Agent 每一步行为可见，透明度高 |
| **底部终端面板** | 执行编译、调试、部署等命令 | ⭐⭐⭐⭐ 省去"切窗口"的上下文成本 |
| **右侧内置浏览器** | 前端项目实时预览 + 元素审查 | ⭐⭐⭐⭐⭐ 修改代码后自动刷新，所见即所得 |

> **实际体验感受**：花样很多很多。

### 3.5 四种对话模式（权限控制）

ZCode 提供了可视化的权限控制，适合不同场景：

| 模式 | 行为 | 适用场景 |
|------|------|----------|
| **Always Ask** | 每次修改文件前征求用户同意 | 对代码修改需严格把关 |
| **Accept Edits** | Agent 自动修改，用户可逐一审核 | 日常开发 |
| **Plan Mode** | Agent 只出计划不执行 | 复杂任务的方案设计阶段 |
| **Bypass** | Agent 完全自主执行 | 充分信任 Agent 的简单任务 |

> **实际体验感受**：暂无体验。

### 3.6 自定义 Commands（/Commands）

ZCode 支持将常用提示词存为快捷命令，以 `/` 前缀唤起。命令以 `.md` 文件形式存储在 `~/.claude/commands/` 目录下，支持参数传递。

> **实际体验感受**：对于有固定工作流（如代码审查、生成 commit message、写单元测试）的团队来说，这套机制省掉了大量重复输入。

### 3.7 MCP 协议支持

ZCode 支持 MCP（Model Context Protocol），允许 Agent 调用外部工具和服务。你可以在可视化界面中配置 MCP 服务器，让 Agent 具备联网搜索、数据库查询、调用 API 等能力。

> **实际体验感受**：未体验到。
